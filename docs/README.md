# engenharia-de-software

Sistema de Gerenciamento de Tarefas (To-Do List)

# Descrição do Projeto
Este projeto consiste no desenvolvimento de um sistema de gerenciamento de tarefas (To-Do List), criado pensando nos métodos de engenharia de software e usando os conceitos de metodologias ágil.

O sistema permite que usuários criem, visualizem e removam tarefas, possibilitando uma melhor organização das tarefas e priorização de atividades.

# Objetivo
Desenvolver um sistema que demonstre na prática os seguintes tópicos:
- Organização de um projeto de software
- Uso de metodologias ágeis (Kanban)
- Versionamento com GitHub
- Implementação de testes automatizados
- Integração com GitHub Actions
- Gestão de mudanças durante o desenvolvimento

# Escopo Inicial do Projeto
No escopo inicial, o sistema contempla as seguintes funcionalidades:

- Criar tarefas
- Listar tarefas cadastradas
- Excluir tarefas
- Concluir a tarefa

As tarefas possuem informações básicas como:
- Título
- Descrição

---

#  Metodologia Utilizada
Foi adotada a metodologia ágil Kanban, utilizando o GitHub Projects para organização das atividades.

O quadro Kanban contém as colunas:
- A Fazer
- Em Progresso
- Concluído

Este modelo permite visualizar o andamento do projeto, priorizar tarefas e acompanhar o progresso.


# Tecnologias Utilizadas
- Ferramentas: JS, HTML e CSS
- Versionamento: GitHub
- Testes Automatizados:
- Integração Contínua: GitHub Actions
- Editor de Código: Visual Studio Code


# Gestão de Mudanças
Durante o desenvolvimento, foi simulada uma mudança de escopo, adicionando uma nova funcionalidade ao sistema.

Inicialmente, o sistema permitia apenas o cadastro e gerenciamento básico de tarefas.  
Durante o projeto, surgiu a necessidade de adicionar prioridade às tarefas, permitindo melhor organização e tomada de decisão.

Essa mudança foi registrada:
- No quadro Kanban (novo card)
- No histórico de commits
